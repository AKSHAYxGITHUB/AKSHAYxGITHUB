## GitHub Stats

<!--
  WHY YOUR STATS SHOWED 2022 DATA:
  github-readme-stats.vercel.app is a shared public Vercel instance used by
  millions of profiles. It constantly hits GitHub's API rate limit. When
  rate-limited, it falls back to serving a CACHED response — which can be
  months or years old. GitHub's Camo CDN (which proxies all README images)
  also caches aggressively at the edge.

  FIX 1: &v=2 appended to all stat card URLs.
  This makes GitHub's Camo CDN treat it as a brand-new URL it has never seen,
  so it must fetch fresh instead of serving the cached 2022 version.

  FIX 2: cache_seconds=86400 tells the stats service to cache for 24h.
  This reduces API calls from the shared server → fewer rate-limit hits
  → more of your requests get a live response instead of a fallback.

  PERMANENT FIX (highly recommended):
  Self-deploy github-readme-stats on Vercel (free, under 5 minutes):
  https://github.com/anuraghazra/github-readme-stats#deploy-on-your-own
  Your own instance only serves your profile → zero rate limit issues ever.
-->
